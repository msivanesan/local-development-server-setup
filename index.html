Example interface: enp0s3

Edit Netplan

sudo nano /etc/netplan/01-network-manager-all.yaml

Paste (change interface if needed):

network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s31f6:
      dhcp4: false
      addresses:
        - 192.168.5.25/24
      gateway4: 192.168.5.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

Apply:

sudo netplan apply

Verify:

ip a

6.2 Enable SSH (CLI Access)

6.3 Install GUI (XFCE) + XRDP

sudo apt install xfce4 xfce4-goodies xrdp -y

Enable XRDP:

sudo systemctl enable xrdp
sudo systemctl start xrdp

Allow RDP port:

sudo ufw allow 3389/tcp
sudo ufw reload

6.4 Configure XRDP to Use XFCE (IMPORTANT)

Per-user XFCE session (REQUIRED for every user)

Run as that user:

echo "startxfce4" > ~/.xsession
chmod +x ~/.xsession
rm -f ~/.Xauthority

System-wide XRDP Fix (ONE-TIME)

Edit XRDP startup script:

sudo nano /etc/xrdp/startwm.sh

Comment this line:

# exec /bin/sh /etc/X11/Xsession

Add below it:

startxfce4 &

Restart XRDP:

sudo adduser xrdp ssl-cert
sudo systemctl restart xrdp

  3. Computer: 192.168.5.25
  4. Click Connect

XRDP Login:

6.6 Fix “Oh no! Something has gone wrong” (NEW USERS)

This happens when .xsession is missing.

Fix:

sudo -u devuser bash
echo "startxfce4" > ~/.xsession
chmod +x ~/.xsession
rm -f ~/.Xauthority
exit
sudo systemctl restart xrdp

Apply to All Future Users (Recommended)

sudo nano /etc/skel/.xsession

Add:

startxfce4

6.7 Disable Sleep / Suspend (SERVER MODE)

Disable system sleep completely

sudo systemctl mask sleep.target suspend.target hibernate.target hybrid-sleep.target

Disable idle power actions

sudo nano /etc/systemd/logind.conf

Set:

HandleSuspendKey=ignore
HandleHibernateKey=ignore
HandleLidSwitch=ignore
HandleLidSwitchExternalPower=ignore
IdleAction=ignore
IdleActionSec=0

Apply:

sudo systemctl restart systemd-logind

Disable XRDP idle timeout

sudo nano /etc/xrdp/sesman.ini

Set:

KillDisconnected=false
DisconnectedTimeLimit=0
IdleTimeLimit=0

Restart:

sudo systemctl restart xrdp

Disable Screen Blanking (XFCE)

Run as XRDP user:

xset -dpms
xset s off
xset s noblank

Make permanent:

nano ~/.xprofile

Add:

xset -dpms
xset s off
xset s noblank

6.8 Prevent SSH Idle Disconnect (Optional)

sudo nano /etc/ssh/sshd_config

Add:

ClientAliveInterval 60
ClientAliveCountMax 0

Restart SSH:

sudo systemctl restart ssh

6.9 Safe Reboot with XRDP Users Logged In

Check sessions:

loginctl

Terminate XRDP user:

sudo loginctl terminate-user devuser

Force reboot (admin):

sudo reboot -f

🚀 STEP 17 — PRODUCTION DEPLOYMENT (SYSTEMD SERVICE)

For production, instead of running Gunicorn in a tmux session, use a systemd service to ensure it starts on boot and restarts if it crashes.

1. Create the Service File

sudo nano /etc/systemd/system/gunicorn-flask.service

2. Paste Configuration

[Unit]
Description=Gunicorn instance to serve Flask app
After=network.target

[Service]
User=devadmin
Group=devadmin
WorkingDirectory=/srv/dev/flask/dataweave
Environment="PATH=/srv/dev/flask/dataweave/venv/bin"
# Standard ExecStart
ExecStart=/srv/dev/flask/dataweave/venv/bin/gunicorn -b 127.0.0.1:5001 app:app

# WebSocket ExecStart (Eventlet)
# ExecStart=/srv/dev/flask/dataweave/venv/bin/gunicorn --worker-class eventlet -w 1 -b 127.0.0.1:5001 app:app

[Install]
WantedBy=multi-user.target

3. Enable and Start Service

sudo systemctl daemon-reload
sudo systemctl enable gunicorn-flask.service
sudo systemctl start gunicorn-flask.service

4. Management Commands

Action Command
Check Status sudo systemctl status gunicorn-flask.service
Restart sudo systemctl restart gunicorn-flask.service
View Logs journalctl -u gunicorn-flask.service
